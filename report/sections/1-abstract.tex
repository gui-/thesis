\begin{abstract}

Programs are usually released without documentation. Therefore, when the documentation is poor, obsolete, or absent, the remaining option is to study the program itself. Over the years, many tools have been proposed to write, document, and comprehend a program. This report presents a survey of systems which provide some of those tools and proposes some innovative tools to improve the current state of the art.

\end{abstract}
